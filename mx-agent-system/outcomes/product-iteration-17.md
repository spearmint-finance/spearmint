# Product Iteration 17 — CONTRIBUTING.md + Screenshots Directive

**Date:** 2026-03-23
**Focus:** Create CONTRIBUTING.md and file screenshots directive — product-owned adoption enablers

## What Happened

Identified two adoption gaps that the product team can directly address:
1. No CONTRIBUTING.md — blocks community engagement
2. No screenshots in README — visitors can't evaluate the app visually

## Deliverables

| Deliverable | Type | Detail |
|-------------|------|--------|
| CONTRIBUTING.md | Product-owned document | Setup instructions, project structure, development workflow, conventions, architecture notes |
| GH#253 | Directive (P2) | Add 3-5 screenshots to README showing dashboard, transactions, accounts, reports, settings |

## Rationale

These are two of the most impactful assets for open-source adoption:
- **CONTRIBUTING.md** is expected by potential contributors — its absence signals the project isn't open to community involvement
- **Screenshots** are the #1 factor in GitHub repo "window shopping" — visitors decide to star/clone within seconds based on visual impressions

Both directly serve the north star metric (GitHub stars).

## Metrics

- Stars: 0 → 0
- Directives: 12 open → 13 open (filed #253 for screenshots)
- human_intervention: no
- measurable_outcome: yes (CONTRIBUTING.md created, directive filed)
